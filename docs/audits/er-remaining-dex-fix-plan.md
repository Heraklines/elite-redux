# ER remaining-dex audit — FIX PLAN / handoff

Generated from the rigorous per-field audit sweep of all remaining ER abilities
(carried by 10+ species) and moves (learned by 37+ mons) — **938 items, 287
findings**. Each move/ability was checked field-by-field (base power, accuracy,
type, category, PP, priority, every flag/tag, every secondary effect, and the
description) against the ER 2.65 dex.

This doc is a **work queue** — pick a section, fix, verify headlessly, add a
regression test + in-game scenario, commit. Do NOT deploy to staging without
the maintainer's OK (testers may be live).

## 🔴 Maintainer rulings (apply these)

1. **Numeric conflicts — the 2.65 dex (`er-moves.ts`) WINS.** 173 findings are
   cases where `init-elite-redux-c-source-corrections.ts` (and, less often, the
   numeric rebalance pass) pins an ER **C-source** value that DISAGREES with the
   `er-moves.ts` dex value for power / PP / accuracy / etc. The maintainer ruled the
   **dex value is authoritative** → make the runtime match `er-moves.ts`. The usual
   lever is to REMOVE or correct the offending entry in
   `init-elite-redux-c-source-corrections.ts` (or ensure the numeric pass in
   `init-elite-redux-vanilla-rebalance.ts` applies the dex value and isn't reverted).
   ⚠️ Watch the global PP cap (20) and any clamps noted in the findings.
2. **Genuine implementation gaps — fix ALL 114.** Missing/changed effects,
   missing flags/tags, and systemic bugs. Fix in batches with headless `expect`
   verification + a vitest regression test each.

**Standing rules** (see CLAUDE.md): dex text overrides parsed flags/vanilla; every
combat-observable fix gets a headless-runner green `expect`, a vitest test, and an
in-game `scenarios.ts` scenario. tsc must stay at baseline. Pass ER ability/move by
numeric runtime id (`ER_ID_MAP.abilities[erId]` / `.moves[erId]`).

## ⭐ Systemic bugs — fix once, resolves many

- **Baseline type-boost stacks with the low-HP 1.5× boost (→ 1.8×).** Overgrow (65),
  Blaze (66), Torrent (67), Swarm (68): ER adds an ALWAYS-ON `MoveTypePowerBoostAbAttr
  (TYPE, 1.2)` (`addBaselineTypeBoost`, `init-elite-redux-vanilla-rebalance.ts`) that
  stacks with the vanilla `LowHpMoveTypePowerBoostAbAttr` (×1.5, gated HP≤1/3), giving
  **1.2×1.5 = 1.8× below 1/3 HP** where the ER C-source is a mutually-exclusive
  `≤1/3 ? 1.5 : 1.2`. **Fix once:** gate the baseline 1.2× to HP>1/3 (or make the two
  attrs mutually exclusive) — one shared fix for all four (and any other ability using
  `addBaselineTypeBoost` alongside a low-HP boost).
- **`c-source-corrections` overriding dex numerics** — see ruling #1; the same lever
  (correct/remove the override) resolves the whole numeric-conflict section.
- Other repeated notes to skim for shared fixes: Toxic-Terrain interactions,
  offense-side status procs, powder-move immunity, crash-damage recoil scaling.
