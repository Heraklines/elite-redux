# Local Asset Pack - players source the ER art themselves

Status: DESIGN APPROVED FOR BUILD (not yet implemented). Staging-first; prod
only with explicit maintainer approval, after the importer is proven.

## Goal

The game stops hosting and distributing any Elite Redux owned art. Players
obtain the ER assets themselves (auto-fetched by their own browser from
public sources the ER team operates or tolerates, or via a pack file shared
in the private community). All online features (cloud saves, ghost teams,
usage tiers, cross-play) are untouched: the save worker stores gameplay
data only, never art.

## What is actually ER-owned (scope)

Only a subset of er-assets is ER's copyright. The rest is vanilla PokeRogue
art (Game Freak's, hosted publicly by PokeRogue itself) which the ER team
has no claim over and which keeps its current hosting + pin.

ER-owned, must move to local supply:

| er-assets path | Content |
|---|---|
| `images/pokemon/elite-redux/{slug}/front|back.{png,json}` | All ER custom species + Redux form battle atlases |
| `images/pokemon/black/elite-redux/{slug}/...` | Black-shiny derivatives of the above (4,053 atlases, generated by us FROM their art, so same exposure) |
| `images/elite-redux/**` | ER UI/trainer art |
| `battle-anims-er/**` | ER battle animation assets ripped from the ROM |
| ER-custom icons/cries where present | inventory step in the build script |

NOT ER-owned (no change): numeric species atlases (`{id}.json`), vanilla
icons sheets, vanilla UI, vanilla audio, `black/{id}` vanilla black-shiny
atlases (derived from Game Freak art, not ER's).

Also leaving er-assets regardless (our data, just needs a new home):
- `usage-tiers.json` -> a tiny public data-only repo (or the game site);
  the nightly worker cron commits there instead.
- Ghost-battle BGM -> ships inside the pack (it is Pokemon music, not ER's,
  but it should not be on our public CDN either).

## Sources, ranked

1. **ForwardFeed/ER-nextdex `static/sprites/`** (public GitHub, GPL-3 repo,
   community dex the ER team tolerates and links). Per-form PNGs with
   regular names: `ABRA_REDUX.png`, `ABRA_REDUX_BACK.png`, `*_SHINY`,
   `*_MEGA*`. Complete coverage of customs, Redux forms and megas.
   **This is the AUTO-FETCH source.** The player's browser downloads
   straight from jsDelivr/raw.githubusercontent; we distribute nothing.
2. **Elite-Redux/eliteredux decomp** (their own public repo). Strongest
   posture (they publish it themselves) but GBA-format graphics need
   palette + animation-table conversion. Phase 3 option, not needed if
   nextdex stays up.
3. **Pack zip built from er-assets** (exact current atlases, including the
   black-shiny set and BGM). Built by a maintainer script, shared ONLY in
   the private Discord. Full fidelity; also the fallback if public sources
   vanish.

## Architecture

### 1. Manifest (in the game repo - data, no art)
`src/data/elite-redux/er-asset-pack-manifest.ts` (generated by script):
for every required game path, its source mapping + flags.

```ts
{ path: "images/pokemon/elite-redux/abra-redux/front", nextdex: "ABRA_REDUX", required: true }
{ path: "images/pokemon/black/elite-redux/abra-redux/front", packOnly: true, required: false }
```

`packVersion` increments when the asset set changes; the importer re-syncs
deltas only.

### 2. Local store
Cache Storage bucket `er-local-assets-v{N}` (survives normal cache wipes
better than HTTP cache, holds multi-GB, streamable). IndexedDB key
`erAssetPackState` tracks installed packVersion + per-entry status.

### 3. Ingestion - two paths, same store
- **Auto-fetch**: for each manifest entry with a `nextdex` mapping, fetch
  the PNG(s) from `cdn.jsdelivr.net/gh/ForwardFeed/ER-nextdex@<pin>/static/sprites/`,
  then synthesize the TexturePacker-style JSON in JS (single-frame atlas:
  frame = whole image; the game's loader already accepts the flat `frames`
  format). Store both halves under the game path. Throttled, resumable,
  progress bar. ~2-3k small files, a few minutes on a normal connection.
- **Zip import**: file picker -> unzip in-browser (`fflate`, streaming) ->
  validate paths against the manifest prefix allowlist -> store. Covers
  the black-shiny extras + BGM + anything auto-fetch cannot supply.

### 4. Serving: service worker interception
The existing service worker gains a fetch handler: requests whose URL path
starts with an ER-owned prefix are answered from `er-local-assets` (404 ->
fall through to network while migration is still on). Zero worker
invocations, zero CDN bandwidth for ER art once installed.

### 5. First-launch gate
If `erAssetPackState.packVersion < required`, the title screen shows an
"Install game assets" panel: [Auto-download] [Import pack file] with
progress + pause/resume. The game refuses to enter a run without the
required set (menus still work). Black-shiny art is optional tier: without
it, blacks fall back to the obsidian tint (the code already does this).

### 6. Black shinies
Distributed in the Discord pack (they are derivatives of ER art; we do not
put them anywhere public). Auto-fetch-only players get tint fallback until
they import the extras pack. Phase 3 option: port the generator to run
in-browser at import time so the pack is unnecessary (heavy: numpy-style
image ops; only if ever needed).

### 7. Build tooling (maintainer side)
`scripts/asset-tooling/local-pack/`:
- `build_manifest.py`: walks er-assets, classifies ER-owned prefixes, maps
  slugs to nextdex filenames (report unmatched for manual mapping), emits
  the manifest TS + a JSON sidecar.
- `build_pack.py`: zips the ER-owned tree (+ black + BGM) into
  `er-asset-pack-v{N}.zip` for Discord distribution.

## Rollout

1. **Phase 1 (staging)**: importer + SW behind `VITE_LOCAL_ASSETS=1`,
   network fallback still on. Test team verifies sprites identical.
2. **Phase 2 (staging default-on)**: first-launch gate active, network
   fallback still on. Bake for a week of testing.
3. **Phase 3 (cutover, NEEDS EXPLICIT MAINTAINER APPROVAL, includes prod)**:
   rehome usage-tiers.json + BGM, deploy prod with importer, flip er-assets
   PRIVATE, purge jsDelivr cache. Public footprint of ER art = zero.

## Risks / notes

- nextdex sprite naming vs our slugs will not match 1:1 everywhere; the
  manifest builder reports unmatched entries and we hand-map the tail.
- nextdex PNGs are single frames; our existing atlases for some customs may
  contain multiple animation frames. Single-frame is an acceptable visual
  downgrade for auto-fetch users; the Discord pack keeps full fidelity.
- If ER-nextdex disappears, auto-fetch breaks for NEW installs only
  (installed players keep their local store); the zip path always works,
  and the decomp-conversion path exists as a deeper fallback.
- iOS Safari storage eviction: Cache Storage can be purged under disk
  pressure; the importer detects missing entries on boot and offers a
  re-sync rather than crashing.
