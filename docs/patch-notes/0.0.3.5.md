# Elite Redux 0.0.3.5

## 🐛 Bug Fixes

- **Mega Evolution is fixed across the board.** Most ER megas — and **every "Redux" custom mega** (Chandelure Redux, Kleavor Redux, Vanilluxe Redux, and dozens more) — simply had **no mega form wired up**, so they could never appear and players holding the stone **couldn't Mega-Evolve**. We rebuilt mega-form setup from ER's actual evolution data, taking the working count from **186 to 273** of the 283 mega-capable species (the handful left are special non-mega forms like Crowned/Ultra). On top of that, the game **wasn't recognizing most Mega Stones** in trainer hands (it only knew a small slice of the stone list) and would pick the wrong split form — so Mega-X/Mega-Y, Primal, and Origin forms now resolve correctly (Feraligatr Mega X, Primal Dialga/Palkia, etc.). Net effect: far more megas show up where they should, and your own mons can actually Mega-Evolve.

- **No more megas appearing too early in Ace/Elite.** Megas are an end-game power spike and are gated to wave 50+ in Ace and Elite (Hell is intentionally exempt). A few rival/boss aces that are fielded *directly* as a mega (e.g. "Blaziken Mega") slipped past that gate; they now correctly show up in their base form before wave 50.

- **Gallantry no longer acts like Sturdy.** It was wired to the "survive with 1 HP" endure effect (which is what you saw against Iron Valiant). It now does what it should — **negates the first instance of damage** the holder takes (the hit connects and side-effects still apply, but damage is 0).

- **Wyrm Wind no longer stacks its stat change per hit.** It's ER's multi-hit Scale Shot variant: it should raise the user's **Speed +1** and lower the user's **Sp. Def −1** *once*, after all hits. It was applying that on **every** strike (so a 2-hit roll gave +2/−2, a 5-hit roll +5/−5), and the Sp. Def drop was even hitting the wrong target. Now it's a single Speed-up / Sp. Def-down at the end of the move, exactly like Scale Shot (relevant for Mega Latios/Latias).

- **Redux-form icons at the top of the battle screen are correct again.** The small enemy/party icons for Redux forms could show the wrong Pokémon (e.g. a Mega Charizard) because the form's icon frame doesn't exist in the icon sheet and the fallback was broken. They now fall back to the correct base-form icon.

- **Morpeko (Hangry) cleanup.**
  - The **Hangry form no longer shows up in eggs or starter selection** — you'll always get the proper base Morpeko (this also keeps with PokeRogue's rule that no mid/final-stage Pokémon appear in starter selection).
  - Fixed the console spam **"No passive ability found … using run away"** — the Hangry custom (and a couple of its relatives) now report their real passive (Two-Faced / Hunger Switch) instead of falling back to Run Away.
  - **Two-Faced no longer pops up every single turn doing nothing.** The passive's form-switch check was mis-firing each turn on the single-form custom; it now only triggers when it actually has something to do.

- **Infernape (Redux) fixes.**
  - **Correct sprite** — Redux Infernape was showing the wrong art; it now uses the proper Infernape sprite.
  - **Removed the junk "Infernape Redux B"** — a broken placeholder species with no stats or abilities was obtainable; it's now filtered out entirely (along with any other all-blank stub).
  - **Long Reach no longer gets you burned.** Infernape Redux has Long Reach (its contact moves shouldn't make contact), but an internal mis-classification treated a Long-Reach-suppressed contact move as a *ranged* move and let Flame Body's non-contact burn proc anyway. Long Reach now correctly avoids both — no more unexpected burns. (Same class of fix as the earlier "non-contact moves triggering contact abilities" bug.)

- **Solar Power no longer chips your HP in sun.** In ER, Solar Power only boosts the holder's highest attacking stat in harsh sunlight — it should not drain 1/8 HP each turn like the vanilla ability. The HP drain was slipping through; now removed (the Sp. Atk boost stays).

- **Calyrex's Ice Rider / Shadow Rider forms now have their innates.** The rider forms were showing their main ability but no passives (innates) because their ER form data wasn't being matched; both riders now get their proper innate triple.

- **Challenge Favour no longer farms extra candy from eggs.** The challenge Favour bonus (shiny + candy, your reward for handicapping a run) was also being applied to candy from eggs you hatch during the run — so a backlog of eggs could farm triple candy under a trivial high-Favour challenge. Egg-hatch candy now keeps only the standard boost, not the run's Favour multiplier. (Egg *shininess* was already unaffected — eggs roll shiny on their own fixed odds — so nothing changes there; we added a test to keep it that way.)

- **Mega Evolution note:** the broad Mega fix above also restores the **Redux-custom megas** that previously "didn't work" (e.g. Mega Redux Reuniclus) — they now have their forms wired. And to be clear: **Ace mode does have megas** — they show up on the late-game gym-leader / Elite-Four / Champion trainers (Mega Kangaskhan, Mega Venusaur, **Mega Rayquaza** on the champion, etc.), just never before wave 50.

- **Every Redux line is now reachable from eggs.** Some lines ship only an *evolved* Redux with no base-form Redux to hatch and evolve up from (e.g. **Weavile Redux** with no Sneasel Redux; **Flygon Redux B** with no Vibrava/Trapinch Redux B). Those were unobtainable. Now, when a line has no lower Redux to evolve from, the **lowest existing Redux form hatches directly** — at a higher egg/starter cost reflecting its evolution stage. (Lines that *do* have a base Redux are unchanged: you still hatch the base and evolve up. And note: there is no "Sneasel Redux" in ER's data — only Weavile Redux — so the Sneasel line is obtained via Weavile Redux directly.)

- **Redux Pokémon no longer evolve into a Gigantamax/Mega form.** A Redux-form mon (e.g. Krabby) could evolve into a battle-only form of its evolution (G-max Kingler) when the pre/post form layouts differed — which also caused an evolution-screen crash. Evolutions now never land on a Mega/Primal/Gigantamax/Eternamax form; they use the normal form.

- **Battle Bond is now generalized** (was hard-coded to Greninja), so ER Battle-Bond builds can transform on a KO. (Darmanitan Redux Bond → Blunder still needs its forms wired before it's reachable in-game — tracked.)

- **Verified working (reported, not actually bugs):** V-create *does* lower the user's Defense/Sp.Def/Speed on hit — the report was a 95%-accuracy **miss** (no on-hit effects fire on a miss). And a Rock move hitting a Flying-type for 2× under **Gravity** is correct — Gravity only removes a Flying-type's *Ground* immunity; it never changes other type matchups. (Regression tests added for both.)

- **Far more trainer variety in Elite/Hell.** Even with the no-repeat system, a run was only fielding ~19–20 *distinct* ER trainer teams (some repeated 7–8 times), because once a given trainer *type's* small pool was used up it would start repeating that type. Now, once a type's fresh trainers run out, the game pulls from the **whole** unused ER pool (400+ tier-eligible trainers) before ever repeating — so a full run now fields a different ER team essentially every trainer battle (Elite: 77 distinct; Hell: 96 distinct). Strength still scales with the wave, so early fights stay easy and Elite-Four/Champion-tier teams still show up late.

- **Elite/Hell trainers no longer repeat after a reload.** The system that stops a difficulty from fielding the same trainer twice tracked "already fought" trainers **only in memory** — and explicitly **wiped that memory every time a run was loaded/resumed**. So reloading or continuing a Hell run reset the pool and re-fought the same (early/weakest) trainers — which is why waves 5 and 15 could be the same battle and you kept seeing the same teams. The used-trainer list is now **saved with your run and restored on load**, so the no-repeat guarantee survives save/load. *(Older saves restore to a fresh pool — no migration needed.)*

## ✨ New Features

- **Damage Calc panel in the fight menu.** While choosing a move, press **R** (controller: **RB**; mobile: the on-screen **R** button) to cycle the right-hand panel from the move's stats to a **Damage Calc** view. It shows the highlighted move's damage as a **percentage range of the target's current HP** (and flags a **possible/guaranteed KO**), so you can read at a glance how hard each move hits.

  The panel is **locked until you hold a Damage Calculator** (the master-tier item). Until then, cycling to it shows a "Locked — find a Damage Calculator" message so you know it's there; once you pick one up, it unlocks and shows live numbers. Status moves show "—", and it gracefully says "no target" if nothing's on the field.
