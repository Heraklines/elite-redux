# Elite Redux v2.65 — Ability/Move Archetype Taxonomy

> **Generated by:** `pnpm run er:cluster-archetypes` + hand-curation.
> **Purpose:** Phase C (behavior port) implements each archetype ONCE as a
> parameterized `AbAttr` / `MoveAttr` subclass; individual abilities/moves
> configure it via data, not new classes.
>
> **Coverage:** ~70% of the 736 custom abilities + 187 custom moves map to
> the archetypes below. The remaining ~280 entities have genuinely novel
> mechanics (the "long tail") and will get bespoke implementations in
> Phase C.

## Methodology

1. Read `src/data/elite-redux/er-abilities.ts` and `er-moves.ts`, filter to
   `archetype === "unknown"` entries (the 736 + 187 ER customs).
2. Normalize each description: lowercase, strip numerals/percentages, replace
   type/stat/status/weather/terrain/hazard names with placeholders, collapse
   synonymous phrasings (`below 1/3 HP` / `under 1/3 HP` / `at low HP`
   → `LOW_HP`, etc.).
3. Group by normalized string. Multi-entry clusters become archetype
   candidates directly; singletons are inspected via overlapping keyword
   buckets (`type-damage-boost`, `entry-misc`, `chance-status-on-hit`, …)
   that reveal larger semantic families.
4. Hand-curate the buckets into ~25 archetypes. Some buckets collapse
   together (e.g., the four entry-setter buckets merge into `entry-effect`
   with a payload union); some split where the implementation is
   meaningfully different (e.g., type-damage-boost vs. flag-damage-boost).

Re-run `pnpm run er:cluster-archetypes` after any change to the description
data to regenerate the raw clusters and bucket counts. The script prints
all multi-entry clusters (~33 of them), a cluster-size histogram, the
keyword-bucket counts, and a sample of unbucketed (long-tail) entries.

Raw script output (cluster report + summary) is the source of truth for
the counts cited below.

## Archetypes (ordered by frequency)

### 1. `composite-vanilla-mashup` (~200 abilities, 0 moves)
**Shape:** `"X + Y."` or `"X + Y + Z."` — a Pokémon's ability is literally
the union of two-or-three named other abilities. This is ER's signature
"layer existing effects" pattern.
**Parameters:** `parts: AbilityId[]` (2 or 3 references to other ER
abilities). Behavior dispatches to each part's archetype in order.
**Examples:**
- `As One` — "Unnerve + Chilling Neigh."
- `Rock Armor` — "Rocky Exterior + takes 10% less damage from attacks."
- `Slime Mold` — "Sticky Hold + Gooey."
- `Smoldering Wood` — "Raw Wood + Flame Body."
- `King of the Jungle` — "Infiltrator + deals 1.5x more damage to Grass-types."
- `Two-Faced` — "Hunger Switch + Elec and Dark deal 1.35x with 10% recoil."

**Phase C class:** `CompositeAbAttr` — composes a list of other `AbAttr`s,
forwarding every hook to each child in registration order. Implemented
**last** in Phase C (depends on every other archetype being available so
the composition is meaningful).

> Note: a handful of "X + Y" descriptions append a free-text rider
> ("…+ takes 10% less damage from attacks"). For those the data wiring
> uses `CompositeAbAttr` for the named parts AND an inline archetype-config
> for the rider clause.

---

### 2. `flag-tagged-move` (~209 moves, 0 abilities)
**Shape:** Move's `longDescription` ends with a category suffix like
`"… Mega Launcher boost"`, `"… Strong Jaw boost"`, `"… Iron Fist boost"`,
`"… Keen Edge boost"`, `"… Mighty Horn boost"`, `"… Archer boost"`,
`"… Hammer-based"`, `"… Striker"`, `"… Air-based"`, `"… Sound-based"`,
`"… Bone-based"`, etc. This is purely a **flag**; the boost itself is an
*ability*-side multiplier on the matching flag. The move's behavior is
its base damage/effect plus the flag bit.
**Parameters:** Already encoded in the data via `MoveFlag` bits; this
"archetype" is recognized at extraction time and folded into the existing
`flags: number[]` array. No new `MoveAttr` class required.
**Examples (moves):**
- `Aqua Fang` — "Bites with aquatic fangs. 20% flinch chance. **Strong Jaw boost.**"
- `Plasma Pulse` — "Double damage against status. 10% paralyze chance. **Mega Launcher boost.**"
- `Frost Brand` — "Slashes with an ice-cold cryo blade. 10% frostbite chance. **Keen Edge boost.**"
- `Femur Breaker` — "The user strikes the foe's legs. Always paralyzes. **Hammer-based.**"

**Phase C class:** none. Flag extraction is data-only — happens in the
existing `extract-fixtures.mjs` / `build-pokerogue-data.mjs` pass. The
*ability*-side multipliers (e.g., Strong Jaw → +30% Bite damage) are
covered by archetype #4 below (`flag-damage-boost`).

---

### 3. `type-damage-boost` (~30 abilities, a few moves)
**Shape:** `"Boosts X-type moves by Nx."` Optionally with an HP threshold
clause: `"… or Mx when below 1/3 HP."` Several sub-shapes converge to the
same archetype with different parameter combos.
**Parameters:**
- `type: PokemonType`
- `multiplier: number`
- optional `lowHpMultiplier?: number` (paired with `lowHpThreshold`,
  default `1/3`)
- optional `recoilPct?: number` (for the "but have N% recoil" variant)
**Examples (clusters merged):**
- `Vengeance`, `Hellblaze`, `Riptide`, `Forest Rage`, `Purgatory`,
  `Earthbound`, `Psychic Mind`, `Rockhard Will`, `Foul Energy`,
  `Flock`, `Short Circuit`, `Fighter` — `"Boosts X-type moves by 1.2/1.3x, or 1.5/1.8x when below 1/3 HP."` (one cluster of 13)
- `Electrocytes` — "Boosts the power of Electric-type moves by 1.25x."
- `Combustion` — "Boosts the power of Fire-type moves by 1.5x."
- `Electric Burst`, `Doom Blast`, `Infernal Rage` — `"X-type moves deal Nx damage but have P% recoil."` (separate sub-cluster)
- `Antarctic Bird` — "Ice-type and Flying-type moves get a 1.3x power boost." (multi-type variant: `types: PokemonType[]`)

**Phase C class:** `TypeDamageBoostAbAttr` with optional `lowHp` and
`recoil` payload.

---

### 4. `flag-damage-boost` (~15-20 abilities)
**Shape:** `"Boosts the power of FLAG moves by Nx."` Equivalent to
archetype #3 but keyed on a `MoveFlag` rather than a type. Includes both
vanilla flags (PUNCHING_MOVE, BITING_MOVE, SOUND_MOVE) and ER-specific
flags (HAMMER_BASED, ARROW, MEGA_LAUNCHER, KEEN_EDGE, MIGHTY_HORN,
STRIKER, IRON_FIST, AURA_BASED, AIR_BASED, BONE_BASED, DANCE).
**Parameters:** `flag: MoveFlag`, `multiplier: number`, optional
`secondaryEffect?: {…}` (some flag boosters also add a status proc).
**Examples:**
- `Keen Edge` — "Boosts the power of slashing moves by 1.3x."
- `Striker` — "Boosts the power of kicking moves by 1.3x."
- `Archer` — "Boosts the power of arrow moves by 1.3x."
- `Mighty Horn` — "Boosts the power of horn and drill-based by 1.3x."
- `Super Slammer` — "Boosts the power of hammer and slamming moves by 1.3x."
- `Giant Wings` — "Boosts the power of wing, wind or air-based moves by 1.3x."

**Phase C class:** `FlagDamageBoostAbAttr`. Composes with `TypeDamageBoostAbAttr`
when an ability does both (rare — handled by `CompositeAbAttr`).

---

### 5. `entry-effect` (~85 abilities, including type-add + setter sub-shapes)
**Shape:** `"On entry, …"` — covers a wide payload variety unified by the
trigger. Sub-shapes (see parameters):
- Set weather (8 turns): `Drizzle`-style customs not yet in vanilla.
- Set terrain (8 turns): `Electro Surge`, plus ER terrain variants.
- Set hazards (Spikes/Toxic Spikes/Stealth Rock/Sticky Web layers).
- Set screen/room (Reflect, Light Screen, Tailwind, Trick Room, Magic
  Room, Gravity, Defense Curl).
- **Add a type** to self (10 entries): `Aquatic`, `Grounded`, `Ice Age`,
  `Half Drake`, `Metallic`, `Phantom`, `Hover`, `Fey Flight`, etc.
- Self stat-boost (5 entries): `Headstrong`, the four `Embody Aspect`
  variants.
- Misc one-shot effects (61 entries, miscellaneous "on entry/switch-in"
  triggers — these are the long-tail entries that still share the trigger
  shape but differ in payload).

**Parameters:** `effect: EntryEffect` discriminated union (see types
below).
```ts
type EntryEffect =
  | { kind: "set-weather"; weather: Weather; turns: number }
  | { kind: "set-terrain"; terrain: Terrain; turns: number }
  | { kind: "set-hazard"; hazard: ArenaTagType; layers?: number }
  | { kind: "set-screen-or-room"; tag: ArenaTagType; turns: number }
  | { kind: "add-self-type"; type: PokemonType }
  | { kind: "self-stat-boost"; stat: Stat; stages: number }
  | { kind: "first-move-priority"; flag: MoveFlag; priority: number }
  | { kind: "scripted-move"; move: MoveId }  // e.g. Jumpscare/Astonish
```
**Examples:**
- `Aquatic` — "Adds Water type on entry." (`add-self-type: Water`)
- `Electro Surge` — "Casts Electric Terrain on entry. Lasts 8 turns." (`set-terrain: ElectricTerrain, turns: 8`)
- `Spider Lair` — "Casts Sticky Web on entry. Lasts 5 turns." (`set-hazard: StickyWeb`)
- `Atlas` — "Sets Gravity on entry for 8 turns." (`set-screen-or-room: Gravity, turns: 8`)
- `Headstrong` — "+1 Spdef on entry." (`self-stat-boost: SPDEF, +1`)
- `Watch Your Step` — "Spreads two layers of Spikes on switch-in." (`set-hazard: Spikes, layers: 2`)
- `Jumpscare` — "Attacks with Astonish on first switch-in." (`scripted-move: Astonish`)

**Phase C class:** `EntryEffectAbAttr` taking the `EntryEffect` discriminator.

### 6. `chance-status-on-hit` (~28 entries — mostly moves)
**Shape:** `"N% chance to STATUS on contact"` or `"… target"` —
proc-status effect on hit. Both attacking moves and on-contact abilities.
**Parameters:** `chance: number` (0-100), `status: StatusEffect`, optional
`onContactOnly?: boolean` (default true for abilities, false for moves
since the move IS the hit).
**Examples (abilities):**
- `Solenoglyphs` — "Biting moves have a 50% chance to badly poison the target."
- `Freezing Point` — "20% chance to get frostbitten on contact and 30% non-contact."
- `Crispy Cream` — "30% to inflict burn/frostbite when hit by contact."
- `Daybreak` — "Burns the foe on contact. Also works on offense."

**Examples (moves):** virtually every flag-tagged move with `"N% X chance"`
in its longDescription — `Aqua Fang`, `Plasma Pulse`, `Bonk` (drowsy),
`Spread Bomb` (burn), `Pebble Shower` (flinch), etc.

**Phase C class:** `ChanceStatusOnHitAbAttr` for abilities; for moves the
existing `StatusEffectAttr` with `chance` already covers this — no new
class needed, just data wiring.

---

### 7. `crit-mod` (~24 abilities)
**Shape:** Modifications to critical-hit behavior — `"Immune to crits"`,
`"+1 crit rate for FLAG moves"`, `"crits ignore abilities and do 2x damage vs resists"`,
etc. Wide payload variety but a single trigger surface (crit roll +
crit-damage application).
**Parameters:**
```ts
type CritMod =
  | { kind: "immune" }
  | { kind: "rate-bonus"; bonus: number; flag?: MoveFlag }
  | { kind: "post-crit-mult"; multiplier: number; condition?: "resists" }
  | { kind: "weakest-defense"; flag?: MoveFlag }
```
**Examples:**
- `Battle Armor`-style ER variants: immune to crits.
- `Fatal Precision` / `Deadly Precision` — "Super-effective moves never miss and always crit."
- `Overrule` — "Crits bypass abilities and do 2x damage vs resists."
- `Deadeye` — "Arrow & cannon moves never miss. Crits hit weakest defense."
- `Bad Omen` — "Foes min roll. Takes 1/4 damage from crits."

**Phase C class:** `CritModAbAttr` with the `CritMod` discriminator.

---

### 8. `damage-reduction-generic` (~30 abilities — split + super-effective + flat)
**Shape:** `"Takes P% less damage from <filter>."` Variants:
- Flat: `"Takes P% less damage from attacks."` (e.g. `Aura Armor`)
- Split-keyed: `"Takes 40% less from Phys. moves."` / `"… Special attacks."`
- Super-effective-keyed: `"Takes 35% less damage from Super-effective moves."`
- Contact-keyed: `"Takes 1/2 dmg from contact moves."`
- Type-keyed (separate archetype #11)
- Weather-conditional: `"Takes 50% less damage if hail is active."` (rare)
**Parameters:** `filter: DamageFilter` (discriminated union), `reduction: number` (0-1).
**Examples:**
- `Permafrost`, `Primal Armor`, `Thick Skin`, `Flame Shield` — `"Takes 35/50% less damage from Super-effective moves."`
- `Dream State`, `Crust Coat` — `"Immune to critical hits. Takes 20% less damage from attacks."` (composite — splits into a `CritMod` + this)
- `Fire Scales`, `Ice Plumes` — `"Halves damage taken by Special moves. Does NOT double SpDef."`
- `Brain Mass` — "Halves damage taken while at full HP."
- `Christmas Spirit` — "Takes 50% less damage if hail is active."

**Phase C class:** `DamageReductionAbAttr` with the `DamageFilter` union.

---

### 9. `stat-trigger-on-event` (~50 entries combined: KO/hit/lower/raise triggers)
**Shape:** `"<event> raise STAT by N stage(s)"` — raises (or lowers)
specific stats on a trigger event. The trigger is the primary axis:
- **On KO** (~7 abilities): `KOs raise STAT by one stage` — `ChillngNeigh`, `Adrenaline Rush`, `Hubris`, `Breezy Neigh`, `Super Strain`.
- **On hit** (~5 abilities): `Inflatable` (`Ups Def and Sp. Def by one stage if hit by Flying or Fire moves`), `Furnace` (`+2 Speed when hit by rocks`).
- **On stat-lowered** (~3-5 abilities): `Narcissist` (`When a stat is lowered, sharply raise both offenses`), `Tactical Retreat`.
- **On foe-raise** (~2): `Egoist` (`Raises its own stats when foes raise theirs`).
- **First-turn / passive** (~30): `Violent Rush`, `Rapid Response`, `Whiteout`, `Ectoplasm`, `Raging Storm` (weather-conditional stat boost), etc.
**Parameters:**
```ts
type StatTrigger = {
  trigger: "on-ko" | "on-hit" | "on-stat-lowered" | "on-foe-stat-raised"
    | "first-turn" | "weather-active" | "low-hp" | "max-hp"
  filter?: { type?: PokemonType; flag?: MoveFlag; weather?: Weather }
  stats: { stat: Stat; stages: number }[]
}
```
**Examples:** see above; many can stack `stats: []` for multi-stat boosts.

**Phase C class:** `StatTriggerOnEventAbAttr` with the `StatTrigger` payload.

---

### 10. `type-conversion` (~13 abilities)
**Shape:** `"Normal moves become X. X moves are empowered."` Some sub-shapes:
- Self type-conversion: `Crystallize` (`Rock-type moves become Ice and get a 1.1x boost`).
- Categorical-flag conversion: `Sand Song` (`Sound moves get a 1.2x boost and become Ground if Normal`).
- Other reflectors / type-coats.
**Parameters:** `sourceType: PokemonType | "any"`, `targetType: PokemonType`,
optional `multiplier?: number`, optional `flag?: MoveFlag` (for the
flag-keyed sub-shape).
**Examples:**
- `Immolate`, `Hydrate`, `Fighting Spirit`, `Spectralize`, `Mineralize`,
  `Draconize` (9-cluster): `"Normal moves become X. X moves are empowered."`
- `Crystallize`, `Superconductor`: `"X-type moves become Y and get a 1.1x boost."`
- `Sand Song`, `Banshee`, `Snow Song`, `Power Metal` (4-cluster): `"Sound moves get a 1.2x boost and become X if Normal."`
- `Reverberate` — "Normal moves are Sound moves." (categorical-only variant)

**Phase C class:** `TypeConversionAbAttr` (probably reusing pokerogue's
existing `MoveTypeChangeAbAttr` with extra parameters).

---

### 11. `type-resist-or-absorb` (~8 abilities)
**Shape:** Two sub-shapes that share the same trigger surface:
- **Resist:** `"Takes Nx damage from TYPE [moves]."`
- **Absorb-heal:** `"Heals N% / 1/4 HP when hit by TYPE."` (often combined
  with `"Redirects TYPE moves. Absorbs them, ups STAT."` for the redirector
  variant.)
**Parameters:** `type: PokemonType | PokemonType[]`,
`effect: { kind: "resist"; multiplier: number } | { kind: "absorb"; healPct: number; redirect?: boolean; statBoost?: {stat: Stat; stages: number} }`.
**Examples:**
- `Elemental Aegis` — "Takes 1/2 damage from Fire, Electric and Water-type attacks."
- `Aegis Ward` — "Takes 1/2 damage from Dark, Ghost and Psychic-type attacks."
- `Fossilized`, `Raw Wood` — "Halves dmg taken by X moves. Boosts own X moves by 1.2x." (composite — splits into this + `TypeDamageBoostAbAttr`)
- `Ice Dew`, `Heat Sink` — "Redirects X moves. Absorbs them, ups highest Atk."
- `Poison Absorb` — "Redirects Poison moves. Absorbs them, healing 25% HP."
- `Mountaineer` — "Immune to Rock-type attacks and Stealth Rock damage."

**Phase C class:** `TypeResistOrAbsorbAbAttr` with the discriminator above.

---

### 12. `type-effectiveness-override` (~10 abilities)
**Shape:** Modifies the type chart — usually grants super-effective hits
against a normally-resistant type, or removes an immunity. ER's
`Dragonslayer`-style design pattern.
**Parameters:** `attackerType: PokemonType`, `targetType: PokemonType`,
`override: number` (the new multiplier).
**Examples:**
- `Fae Hunter`, `Lumberjack`, `Monster Hunter`, `Firefighter` (4-cluster): `"Deals 1.5x damage to X. Takes 0.5x damage from X."` (composite of resist + effectiveness-override)
- `Phantom Pain` — "Ghost-type moves deal normal damage to Normal."
- `Molten Down` — "Fire-type is super effective against Rock-type."
- `Overcharge` — "Electric is super effective vs Electric. Can paralyze Electric."
- `Ground Shock` — "Target Grounds aren't immune to Electric but resist it instead."
- `Bone Zone` — "Bone moves ignore immunities and deal 2x on not very effective."

**Phase C class:** `TypeEffectivenessOverrideAbAttr`. May need to hook
into the existing type-chart resolution (`TypeBoostAttr` / damage calc).

---

### 13. `priority-modifier` (~12 abilities)
**Shape:** `"+N priority for FLAG / TYPE moves"`, possibly conditional on
HP threshold. Multiple sub-shapes:
- HP-gated type-keyed: `Flaming Soul`, `Frozen Soul`, `Early Grave`,
  `Stygian Rush`, `Tidal Rush` (5-cluster) — `"X-type moves get +1 priority at max HP."`
- HP-gated, flag-keyed: `Blitz Boxer` — `"At full HP, gives +1 priority to this Pokémon's punching moves."`
- HP-gated, ability-side: `Volt Rush`, `Pretty Privilege` (2-cluster) —
  `"At full HP, gives +1 priority to its X-type moves."`
- First-move: `On the Prowl` — "+1 priority for the first turn."
- First-entry: `Sidewinder`, `Edgelord`, `Cutthroat` — "The first <flag> move each entry gets +1 priority. Resets on KO."
- Categorical: `Galeforce Wings` — "Flying moves get +1 Priority.", `Higher Rank` — "Priority moves get a 1.2x boost." (boost-side variant)
**Parameters:** `priority: number`, `condition: PriorityCondition` (HP
threshold, first-turn, first-entry, etc.), `filter: { type?, flag? }`.
**Examples:** see above.

**Phase C class:** `PriorityModifierAbAttr` with discriminated condition.

---

### 14. `passive-recovery` (~12 abilities)
**Shape:** `"Recovers N/N (or P%) of max HP at the end of each turn"`, with
optional precondition (asleep, in fog, on field, etc.).
**Parameters:** `healFraction: number`, optional `condition?: HealCondition`
(`sleeping`, `weather: Weather`, `terrain: Terrain`, `default-always`).
**Examples:**
- "Recovers 1/16 of max HP at the end of each turn." (cluster-anonymous; one of the customs)
- `Sweet Dreams` — "Heals 1/8 of max HP every turn if asleep. Immune to Bad Dreams." (composite)
- `Soul Tap` — "Drain 10% HP from foes at the end of each turn in fog." (siphon variant — closer to next archetype, but lumped here)

**Phase C class:** `PassiveRecoveryAbAttr`.

---

### 15. `damage-deal-heal` / `lifesteal` (~10 abilities)
**Shape:** `"Heals N/N of damage dealt"` or `"Heals user for N/N of the damage they deal"`.
**Parameters:** `healFraction: number`, optional `filter?: { type?: PokemonType; flag?: MoveFlag }`.
**Examples:**
- `Energy Siphon` — "Heals the user for 1/4 of the damage they deal."
- `Energy Tap` — "Heals the user for 1/8 of the damage they deal."
- `Soul Eater`, `Scavenger`, `Predator`, `Looter` (4-cluster): `"Dealing a KO heals 1/4 of this Pokémon's max HP."` (technically `ko-heal` — split or merged?)
- `Hydro Circuit` — "Electric moves +50%; Water moves siphon 25% damage." (composite)
- `Life Steal` — "Steals 1/10 HP from foes each turn."

**Phase C class:** `LifestealAbAttr` (covers both per-hit and on-KO heal
via a discriminator: `trigger: "on-hit-deal" | "on-ko"`).

---

### 16. `move-replacement` (~13 abilities)
**Shape:** A specific move learned by this Pokémon is replaced with a
different (often-stronger) move while the ability is active. Also covers
"this move becomes that move under condition X" patterns.
**Parameters:** `replaceMap: Record<MoveId, MoveId>`, optional
`condition?: ReplaceCondition`.
**Examples:**
- `Temporal Rupture` — "Roar of Time is altered drastically."
- `Angel's Wrath` — "Drastically alters all of the users moves."
- `Cold Plasma` — "Electric type moves now inflict burn instead of paralysis." (status-side replacement; technically merges with `type-conversion`)

**Phase C class:** `MoveReplacementAbAttr` (uses pokerogue's existing
move-id remap mechanism — likely a thin wrapper).

---

### 17. `proc-followup-attack` (~10-12 abilities)
**Shape:** `"Triggers N BP <Move> after using a TYPE-type / FLAG / <kind> move."` —
the Pokémon's signature combo: every X move chains into a second free
attack at fixed BP.
**Parameters:** `followup: MoveId`, `followupBp: number`, `trigger:
{ type?: PokemonType; flag?: MoveFlag; tag?: "damaging" | "dance" | "sound" }`.
**Examples:**
- `Volcano Rage` — "Triggers 50 BP Eruption after using a Fire-type move."
- `Thundercall` — "Triggers Smite at 20% power when using an Electric move."
- `Pyro Shells` — "Triggers 50 BP Outburst after using a Mega Launcher move."
- `Frost Burn` — "Triggers 40BP Ice Beam after using a Fire-type move."
- `Glacial Rage` — "Triggers 50 BP Blizzard after using a Ice-type move."
- `Aftershock` — "Triggers Magnitude 4-7 after using a damaging move."
- `Two Step` — "Triggers 50BP Revelation Dance after using a Dance move."
- `Blade Dance` — "Triggers 50 BP Leaf Blade after using a dance move."
- `Frost Dragon` — "Triggers 50 BP Blizzard after using a Dragon or Ice-type move."
- `Purple Haze` — "Triggers a 20BP Poison Gas after using a move."
- `Backflip`, `Break it Down`, `Lunar Wrath` — `"After using <X>, follow up with a NN BP <Y>."`

**Phase C class:** `ProcFollowupAttackAbAttr`. One of the more involved
archetypes — needs to inject a free attack into the turn queue without
breaking PP / accuracy / ability-checks.

---

### 18. `on-hit-counter-attack` (~5-7 abilities)
**Shape:** `"Counters contact with N BP <Move>."` Symmetric to #17 but
*reactive* — fires when this Pokémon is hit, not when it acts.
**Parameters:** `counterMove: MoveId`, `counterBp: number`, `filter:
{ contact?: boolean; type?: PokemonType; flag?: MoveFlag }`.
**Examples:**
- `Cold Rebound` — "Attacks with Icy Wind when hit by a contact move."
- `Clap Trap` — "Counters contact with 50BP Snap Trap."
- `Ice Downfall` — "Counters contact with 60BP Icicle Crash."
- `Ultra Instinct` — "Counters contact with 20BP Vacuum Wave. Takes .8x damage." (composite)
- `Acid Reflux` — "Uses 20BP Acid when it takes damage."
- `Chilling Pellets` — "Uses 13BP Icicle Spear when hit by contact."

**Phase C class:** `OnHitCounterAttackAbAttr`. Shares plumbing with
archetype #17 (both inject a "free move" into the turn).

---

### 19. `multi-hit-override` (~7 abilities)
**Shape:** `"X moves hit twice."` or `"Moves hit N-M times."` — overrides
the move's hit-count.
**Parameters:** `filter: { type?: PokemonType; flag?: MoveFlag; tag?: "all" | "damaging" }`,
`hits: number | [number, number]` (fixed or range),
optional `secondaryHitMultiplier?: number` (e.g. "2nd hit at 40%").
**Examples:**
- `Raging Boxer` — "Punching moves hit twice. 1st hit at 100% power, 2nd hit at 40%."
- `Hyper Aggressive` — "Moves hit twice. Second hit does 25% damage."
- `Primal Maw` — "Biting moves hit twice. 2nd hit does 0.4x damage."
- `Raging Moth` — "Fire moves hits twice, both hits at 70% power."
- `Ice Cold Hunter` — "Ice-type moves hit twice in hail."
- `Dual Wield` — "Mega Launcher and Keen Edge moves hit twice for 70% damage."
- `Unrelenting` — "All attacking moves can hit 2-5 times."
- `Multi-Headed` — "Hits as many times, as it has heads." (form-dependent — likely bespoke)

**Phase C class:** `MultiHitOverrideAbAttr`.

---

### 20. `conditional-damage` (~10 abilities)
**Shape:** `"Doubles damage if <condition>"` / `"x1.5 vs statused"` / `"x2 vs low HP foes"`. Boost gated on a per-target condition.
**Parameters:** `condition: TargetCondition` (`statused`, `confused`, `sleeping`, `low-hp`, `boosted-stats`, `non-type-X`), `multiplier: number`.
**Examples:**
- `Dreamcatcher` — "Doubles damage if an opponent is sleeping. Pursues sleeping foes."
- `Cosmic Daze` — "Deals 2x damage vs confused and enraged foes."
- `Pretty Princess` — "Does 50% more damage if the target has any lowered stat."
- `Dream Invasion` (move) — "Deals 2x damage to sleeping foes. 10% chance for drowsy. Hits both foes."

### 23. `weather-or-terrain-interaction` (~10 entries; merges weather-stat-boost +
terrain-stat-boost + weather-protect)
**Shape:** Behavior gated on active weather/terrain. Sub-shapes:
- Stat boost in weather: `"Ups highest attacking stat by 1.5x in WEATHER."`
- Type-boost in weather: `"Boosts Water moves in rain."`
- Protection: `"Blocks priority and reduces special damage by 1/2 in sand."`
- Status-immunity in weather: `"Protects its side from status and secondary effects in sand."`
- Terrain-gated speed: `"If Grassy Terrain is active, gets a 1.5x Speed boost."`
**Parameters:** `condition: { weather?: Weather; terrain?: Terrain }`,
`effect: WeatherEffect` (stat-boost, type-boost, damage-reduction, status-immunity, …).
**Examples:**
- `Whiteout`, `Ectoplasm`, `Raging Storm` — `"Ups highest attacking stat by 1.5x in WEATHER."` (3-cluster)

- `Desert Cloak` — "Protects its side from status and secondary effects in sand."
- `Sand Guard` — "Blocks priority and reduces special damage by 1/2 in sand."
- `Jungle Fever` — "If Grassy Terrain is active, gets a 1.5x Speed boost."
- `Smokey Maneuvers` — "Evasion is boosted by 1.25x in fog."

**Phase C class:** `WeatherOrTerrainInteractionAbAttr`.

## Phase C implementation order

Work the archetypes in **dependency order** (composite last), starting with
the highest-frequency, lowest-complexity ones to deliver early wins:

1. **`flag-tagged-move`** (#2) — data-only at extraction time. 0 hours
   runtime work; just `er:build` change.
2. **`type-damage-boost`** (#3) and **`flag-damage-boost`** (#4) — both
   slot into the existing `MoveEffectivenessAbAttr` family. Estimated
   ~3-4 hours including tests + wiring 5-10 sample abilities.
3. **`chance-status-on-hit`** (#6) — reuses pokerogue's existing
   `PostDefendContactApplyStatusEffectAbAttr` / `StatusEffectAttr`. Mostly
   data wiring. ~2 hours.
4. **`damage-reduction-generic`** (#8), **`type-resist-or-absorb`** (#11) —
   ~4 hours each (separate trigger surfaces, but symmetrical).
5. **`stat-trigger-on-event`** (#9) — biggest payload union; ~6-8 hours
   for the framework + cover-all-triggers tests.
6. **`entry-effect`** (#5) — large payload union; ~6 hours.
7. **`priority-modifier`** (#13), **`accuracy-mod`** (#21),
   **`crit-mod`** (#7), **`multi-hit-override`** (#19) — ~3 hours each.
8. **`weather-or-terrain-interaction`** (#23) — ~4 hours.
9. **`type-conversion`** (#10), **`type-effectiveness-override`** (#12),
   **`move-replacement`** (#16) — ~4 hours each (touches type-chart and
   move resolution).
10. **`proc-followup-attack`** (#17), **`on-hit-counter-attack`** (#18) —
    most invasive: they inject "free moves" into the turn queue.
    ~8 hours each including extensive tests.
11. **`passive-recovery`** (#14), **`damage-deal-heal`** (#15),
    **`status-immunity`** (#24), **`conditional-damage`** (#20),
    **`pp-effect`** (#22), **`form-change`** (#25) — ~3 hours each.
12. **`composite-vanilla-mashup`** (#1) — last, because it depends on every
    other archetype being available. ~6 hours including disambiguation
    of mashup parts.
