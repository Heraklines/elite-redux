## Accepted Gaps

Gaps below are tracked here per the Phase B plan, with remediation owned by Phase C asset work.

### ER-custom species without upstream sprites

**Status:** structural — the upstream `Elite-Redux/eliteredux` repo only ships PNGs for vanilla
Gen1-9 Pokémon. ER-original species (`*_REDUX`, custom megas) typically have no source PNG.

**Remediation (Phase C):** source sprites from the ER community (Discord asset packs, fan ports),
hand-create placeholders, or render shiny variants from palette files. Track individual sprite
additions here as they land.

### Box icons (synthesised from front sprite)

**Status:** the upstream `Elite-Redux/eliteredux` repo ships `icon.png` for ~1100 vanilla species
but the ForwardFeed/ER-nextdex backfill does NOT include icons. The ~778 species we pull from
the dex (Hisuian forms, Paldean forms, ER-customs, custom megas) would 404 on storage-box and
party-menu icon loads without synthesis. `pnpm run er:synth-icons` downsamples each missing
species' `front.png` 64×64 → 32×32 (nearest-neighbour) and stacks two copies vertically into
the canonical 32×64 two-frame ER icon sprite-sheet. Re-run the synthesiser (it's idempotent)
if the audit reports missing `icon.png` entries.

### Shiny variants (generated from palette files)

**Status:** the manifest expects 6 shiny PNGs per species — 3 tiers (regular/+/++) × 2 views
(front/back). They are GENERATED by `pnpm run er:render-shinies` from the upstream `shiny.pal`
JASC palette file: tier 1 is the exact palette swap, tier 2/3 are derived by hue-rotating the
shiny palette by 120°/240° to give the rare/legendary tiers a distinct dominant hue.
Re-run the renderer (it's idempotent) if the audit reports missing `shiny*.png` entries.

### anim_front + footprint partial coverage

**Status:** ~185 vanilla species have front+back+icon but no animated front or footprint frames.
Acceptable for Phase B — these are optional visual flourishes (footprint is dex-only, anim_front
falls back to static front). Phase C should fill where feasible.
